# Standard proposal template

Use this template for typical features, new services, or moderate refactors.

## Executive summary

Provide a brief summary of the problem, the proposed solution, and the expected
impact. This section should be readable by non-technical stakeholders.

## Problem statement and context

Describe the background and the specific problem you are solving. Explain why
this problem needs to be addressed now.

## Goals and non-goals

List what the proposal aims to achieve and what is explicitly out of scope.

### Goals

- Goal 1
- Goal 2

### Non-goals

## Proposed architecture

Describe the high-level design of the solution.

- Include a Mermaid diagram showing the components and data flow.
- Explain the role of each component.

## Detailed design

Detail the implementation specifics.

### Data models

Describe any new database schemas, file formats, or data structures.

### API design

List new or modified API endpoints, including request and response payloads.

## Alternatives considered

List other solutions you evaluated. For each alternative, explain why it was
rejected.

## Risks and mitigations

Identify potential risks (e.g., security, performance, migration difficulties)
and how you plan to address them.
