# Lightweight proposal template

Use this template for small features, minor changes, or simple refactors.

## Title

Include a clear title, the author, date, and current status (e.g., Draft, Under
Review, Approved).

## Problem and goals

Explain the problem you are solving and what this change will achieve. Keep this
to one or two paragraphs.

## Proposed design

Describe the proposed change.

- Include code snippets or configuration changes if applicable.
- List any new dependencies.
- Explain how the change affects existing systems.

## Alternatives considered

List at least one alternative approach you considered and explain why you did
not choose it.
